# 📤 GitHub 手动上传指南（图文版）

## 🎯 第一步：打开 GitHub 仓库

访问：**https://github.com/MyraWang0406/PSM-DID-uplift**

你会看到一个空仓库页面。

---

## 📦 第二步：准备要上传的文件

在你的电脑上，找到项目文件夹：
```
D:\Download\cursor\AI 导购 渠道质量评判 PSM DID uplift\
```

### ✅ 需要拖拽上传的内容：

#### 📁 文件夹（4个，整个文件夹拖拽）：

1. **`app`** 文件夹
   - 包含：layout.tsx, page.tsx, page.css, globals.css

2. **`components`** 文件夹
   - 包含：Dashboard.tsx, FunnelTable.tsx, UpliftTable.tsx, DIDChart.tsx

3. **`public`** 文件夹
   - 包含：funnel_by_channel.json, uplift_by_channel.json, did_timeseries.json

4. **`.github`** 文件夹（注意前面有点，是隐藏文件夹）
   - 包含：workflows/deploy.yml

#### 📄 文件（5个，单独拖拽）：

5. **`package.json`**
6. **`tsconfig.json`**
7. **`next.config.js`**
8. **`.gitignore`**（注意前面有点）
9. **`README.md`**（可选）

---

## 🖱️ 第三步：上传文件

### 方法 A：一次性上传所有文件（推荐）

1. 在 GitHub 仓库页面，点击 **"Add file"** 按钮
2. 选择 **"Upload files"**
3. 在文件资源管理器中，**按住 Ctrl 键**，依次点击选择：
   - `app` 文件夹
   - `components` 文件夹
   - `public` 文件夹
   - `.github` 文件夹（如果看不到，需要在文件资源管理器中启用"显示隐藏文件"）
   - `package.json`
   - `tsconfig.json`
   - `next.config.js`
   - `.gitignore`
   - `README.md`
4. **拖拽**这些选中的文件和文件夹到 GitHub 的上传区域
5. 等待上传完成

### 方法 B：分批上传（如果方法A不行）

#### 第一批：上传文件夹
1. 拖拽 `app` 文件夹
2. 拖拽 `components` 文件夹
3. 拖拽 `public` 文件夹
4. 拖拽 `.github` 文件夹

#### 第二批：上传配置文件
1. 拖拽 `package.json`
2. 拖拽 `tsconfig.json`
3. 拖拽 `next.config.js`
4. 拖拽 `.gitignore`
5. 拖拽 `README.md`

---

## ✍️ 第四步：提交文件

1. 在页面底部的 **"Commit changes"** 区域
2. 填写提交信息：
   ```
   Initial commit: AI导购效果分析看板
   ```
3. 选择 **"Commit directly to the main branch"**
4. 点击绿色的 **"Commit changes"** 按钮

---

## ⚙️ 第五步：启用 GitHub Pages

1. 在仓库页面，点击顶部的 **"Settings"** 标签
2. 在左侧菜单中找到 **"Pages"**（在 "Code and automation" 部分）
3. 在 "Source" 部分，下拉菜单选择 **"GitHub Actions"**
4. 页面会自动保存

---

## ⏳ 第六步：等待自动部署

1. 点击仓库顶部的 **"Actions"** 标签
2. 你会看到 "Deploy to GitHub Pages" 工作流正在运行
3. 等待 2-5 分钟，直到看到绿色的 ✅ 标记
4. 如果看到红色的 ❌，点击查看错误信息

---

## 🌐 第七步：访问网站

部署成功后，访问：
**https://myrawang0406.github.io/PSM-DID-uplift/**

## 🔍 如何显示隐藏文件夹（.github 和 .gitignore）

### Windows 10/11：
1. 打开文件资源管理器
2. 点击顶部菜单 **"查看"**
3. 勾选 **"隐藏的项目"**
4. 现在可以看到 `.github` 文件夹和 `.gitignore` 文件了

## ⚠️ 常见问题

### Q: 找不到 `.github` 文件夹？
A: 需要启用"显示隐藏文件"，参考上面的说明。

### Q: 上传后 Actions 失败？
A: 检查是否所有文件都已上传，特别是：
- `.github/workflows/deploy.yml` 必须存在
- `package.json` 必须存在
- `public/` 文件夹下的三个 JSON 文件必须存在

### Q: 上传的文件太多？
A: 只上传清单中的文件，不要上传：
- `node_modules/` 文件夹
- `.next/` 文件夹
- `out/` 文件夹
- `*.csv` 文件
- `*.py` 文件

### Q: 如何验证上传成功？
A: 在 GitHub 仓库页面，你应该能看到：
- ✅ `app/` 文件夹
- ✅ `components/` 文件夹
- ✅ `public/` 文件夹
- ✅ `.github/` 文件夹
- ✅ `package.json` 等配置文件
